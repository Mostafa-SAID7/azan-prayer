# 📖 Glossary

Terms and definitions used in Payer documentation.

## Islamic Terms

| Term | Definition |
|------|-----------|
| **Fajr** | Dawn prayer, earliest prayer of the day |
| **Sunrise** | Time when sun appears on horizon |
| **Dhuhr** | Midday prayer, performed around noon |
| **Asr** | Afternoon prayer |
| **Sunset** | Time when sun sets on horizon |
| **Maghrib** | Evening prayer, performed at sunset |
| **Isha** | Night prayer, performed after sunset |
| **Imsak** | Time to stop eating during Ramadan fast |
| **Hijri** | Islamic lunar calendar |
| **Adhan** | Call to prayer (the app uses this audio) |
| **Salah** | Islamic prayer (same as "prayer" in app) |
| **Qibla** | Direction toward Mecca for prayer |

## Technical Terms

| Term | Definition |
|------|-----------|
| **API** | Application Programming Interface - way apps communicate |
| **CORS** | Cross-Origin Resource Sharing - browser security |
| **PWA** | Progressive Web App - web app installable as app |
| **Service Worker** | Background script that enables offline support |
| **Bundle** | Compressed JavaScript files ready to run |
| **Build** | Process of preparing code for production |
| **Cache** | Storing data locally to reduce requests |
| **Minify** | Removing unnecessary characters from code |
| **HMR** | Hot Module Replacement - live code reload |
| **SSR** | Server-Side Rendering - rendering on server |
| **CSR** | Client-Side Rendering - rendering in browser |
| **REST** | Architecture style for APIs |
| **JSON** | Format for data exchange (human readable) |
| **CLI** | Command Line Interface - terminal commands |
| **DOM** | Document Object Model - HTML structure in browser |

## Project Terms

| Term | Definition |
|------|-----------|
| **Smart Component** | Component that manages state and logic |
| **Dumb Component** | Component that only displays props (presentational) |
| **Hook** | JavaScript function that adds React state |
| **Context** | Way to share data across components |
| **Props** | Data passed from parent to child component |
| **State** | Data that changes over time |
| **Render** | Process of displaying component on screen |
| **Virtual DOM** | React's in-memory representation of UI |
| **Rerender** | Component updating on screen when data changes |
| **Lifecycle** | Component birth, life, and death |
| **Mount** | Component first appears on screen |
| **Unmount** | Component removed from screen |
| **Key** | Unique identifier for list items |

## Design Patterns

| Pattern | Definition |
|---------|-----------|
| **Observer** | Component subscribes to state changes |
| **Factory** | Function that creates objects |
| **Singleton** | Only one instance of something exists |
| **Memoization** | Caching results of expensive operations |
| **Composition** | Building complex from simpler components |
| **Props Drilling** | Passing props through many levels |
| **Render Props** | Passing function as prop to control rendering |
| **Higher Order Component** | Function that creates wrapped component |
| **Dependency Injection** | Providing dependencies to functions |
| **Module Pattern** | Encapsulating code in module |

## Styling Terms

| Term | Definition |
|------|-----------|
| **Utility Class** | Single-purpose CSS class (like `text-red-500`) |
| **Tailwind** | Utility-first CSS framework |
| **CSS Variable** | Reusable CSS value (`--primary-color`) |
| **Breakpoint** | Screen width where layout changes (`md:`, `lg:`) |
| **RTL** | Right-to-left text direction (Arabic) |
| **LTR** | Left-to-right text direction (English) |
| **Responsive** | Design that works on different screen sizes |
| **Mobile-first** | Designing for mobile before desktop |
| **Semantic HTML** | Using correct HTML tags (`<button>` not `<div>`) |
| **Accessibility** | Making app usable by everyone (A11Y) |
| **WCAG** | Web Content Accessibility Guidelines |
| **Contrast Ratio** | Difference between text and background colors |
